Chapter 1

Introduction to CSS

What is CSS?

CSS stands for Cascading Style Sheets. It is the language used to control the visual presentation of HTML elements — layout, colour, typography, spacing, and animation.

  • Cascading — multiple style rules can apply to one element; the browser resolves conflicts using specificity and order.
  • Style Sheet — a document (or block) that contains styling rules.
Separation of concerns: HTML defines structure, CSS defines presentation. Keeping them separate makes both easier to read, maintain, and update.

Ways to Add CSS

MethodHowWhen to use
External stylesheet<link rel="stylesheet" href="style.css"> in <head>Always — best practice for any real project.
Internal <style><style> block inside <head>Small single-page demos or email templates.
Inline stylestyle="color: red" attribute on an elementLast resort or dynamic JS overrides. Avoid for layout.
External stylesheets are cached by the browser — users download the CSS once and reuse it across all pages. Inline styles cannot be cached and have very high specificity, making them hard to override.

Anatomy of a CSS Rule

h1 { color: navy; font-size: 2rem; } ↑ selector ↑ property: value ← declaration
  • Selector — targets the element(s) to style.
  • Declaration block — the { } containing one or more declarations.
  • Declaration — a property: value pair ended with ;.
  • Property — the aspect to style (e.g. color, font-size).
  • Value — what to set the property to (e.g. navy, 2rem).

Chapter 3

The Cascade, Specificity & Inheritance

The Cascade

When multiple rules target the same element and same property, the browser uses the cascade to decide which one wins. It considers three things in order:

PriorityFactor
1stOrigin & importance — browser defaults < author styles < !important
2ndSpecificity — how specific the selector is (see below)
3rdSource order — the last rule wins if specificity is equal

Specificity

Specificity is a score calculated from the selector. Higher score wins.

Selector typeSpecificity value
Inline style (style="…")1-0-0-0 (highest)
ID selector (#id)0-1-0-0
Class, attribute, pseudo-class (.class, [attr], :hover)0-0-1-0
Element, pseudo-element (p, ::before)0-0-0-1
Universal selector (*)0-0-0-0 (no specificity)
/* specificity: 0-0-0-1 */ p { color: black; } /* specificity: 0-0-1-0 — wins */ .intro { color: blue; } /* specificity: 0-1-0-0 — wins over both */ #hero { color: red; }
Avoid !important — it overrides everything and makes debugging very hard. Fix specificity issues by restructuring selectors instead.

Inheritance

Some CSS properties are inherited — child elements automatically receive the parent's value. Others are not.

Inherited by defaultNot inherited by default
color, font-family, font-size, font-weight, line-height, text-align, visibilitymargin, padding, border, background, width, height, display, position

You can control inheritance explicitly with these special values usable on any property:

ValueMeaning
inheritForce the property to use the parent's value.
initialReset to the browser's default value for that property.
unsetActs as inherit if the property is inherited, otherwise initial.
Chapter 4

The Box Model

Box Model Layers

Every HTML element is rendered as a rectangular box made up of four layers, from inside out:

LayerPropertyDescription
Contentwidth, heightThe actual content area — text, image, etc.
PaddingpaddingSpace between content and border. Background colour fills padding.
BorderborderA line around the padding and content.
MarginmarginSpace outside the border — transparent, separates elements.
Margin vs Padding: Margin is space outside the element (between elements). Padding is space inside the element (between content and border). Background colour does NOT extend into margin.

box-sizing

ValueHow width/height is calculated
content-box (default)Width/height applies to content only. Padding and border are added on top — total size is larger.
border-boxWidth/height includes padding and border. Content shrinks to fit. Recommended.
/* Set globally — always do this */ *, *::before, *::after { box-sizing: border-box; }

Margin, Padding & Border Shorthand

/* All four sides equal */ margin: 16px; /* top/bottom left/right */ padding: 8px 16px; /* top right bottom left (clockwise) */ margin: 10px 20px 10px 20px; /* Individual sides */ margin-top: 8px; padding-left: 24px; /* Border shorthand: width style colour */ border: 1px solid #ccc; border-radius: 8px;
Margin collapsing: When two vertical margins meet (e.g. bottom margin of one block and top margin of the next), they collapse into a single margin equal to the larger of the two. This only happens with block-level elements in normal flow.

CSS Units

UnitTypeDescription
pxAbsolutePixels — fixed size, not scalable with user font preferences.
remRelativeRelative to the root (html) font size. Default = 16px, so 1rem = 16px. Best for font sizes.
emRelativeRelative to the element's own font size. Compounds when nested — use carefully.
%RelativeRelative to the parent element's value. Common for widths.
vw / vhViewport1vw = 1% of viewport width. 1vh = 1% of viewport height.
chRelativeWidth of the "0" character in the current font. Useful for limiting line length.
frGridFractional unit — only used in CSS Grid. 1fr = one equal share of available space.
rem vs em: Prefer rem for font sizes — it always references the root and avoids the compounding problem of em. Use em when you want spacing to scale proportionally with the element's own font size (e.g. padding: 0.5em).

Chapter 7

Display & Positioning

The display Property

ValueBehaviour
blockTakes full width, starts on a new line. width/height apply.
inlineFlows with text. width/height ignored. Margin/padding only affect horizontal space.
inline-blockFlows with text but respects width, height, all margin/padding.
noneElement removed from layout completely. Takes up no space.
flexBecomes a flex container — children laid out with Flexbox.
gridBecomes a grid container — children laid out with CSS Grid.
display: none vs visibility: hidden: none removes the element from layout (no space taken). visibility: hidden hides it visually but it still occupies space.

The position Property

ValueBehaviour
static (default)Normal document flow. top/left/right/bottom have no effect.
relativePositioned relative to its normal position. Offset with top/left/etc. Still occupies original space.
absoluteRemoved from flow. Positioned relative to the nearest ancestor with position other than static.
fixedRemoved from flow. Positioned relative to the viewport. Stays in place when scrolling.
stickyActs as relative until the scroll threshold is hit, then behaves like fixed.
/* Tooltip pattern: child absolute inside relative parent */ .parent { position: relative; } .tooltip { position: absolute; top: 100%; left: 0; } /* Sticky header */ header { position: sticky; top: 0; z-index: 100; }
z-index controls stacking order — higher value = on top. Only works on elements with position other than static, and on flex/grid children.

Flexbox

Flex Container Properties

Set display: flex on the parent. These properties control how children are arranged.

PropertyValuesEffect
flex-directionrow (default), column, row-reverse, column-reverseSets the main axis direction.
justify-contentflex-start, flex-end, center, space-between, space-around, space-evenlyAligns children along the main axis.
align-itemsstretch (default), flex-start, flex-end, center, baselineAligns children along the cross axis.
flex-wrapnowrap (default), wrap, wrap-reverseWhether children wrap to a new line when they overflow.
gape.g. 16px or 16px 8pxSpace between flex children. Cleaner than margins.
align-contentflex-start, center, space-between, etc.Aligns rows along cross axis when wrapping. Only active when flex-wrap: wrap.

Flex Item Properties

These go on the children (flex items).

PropertyDescription
flex-growHow much the item grows to fill available space. Default 0 (don't grow). 1 = grow equally.
flex-shrinkHow much the item shrinks when space is tight. Default 1.
flex-basisInitial size before growing/shrinking. Default auto.
flexShorthand for grow shrink basis. Common: flex: 1 = 1 1 0.
align-selfOverride align-items for a single item.
orderVisual ordering. Default 0. Lower = appears first.
.nav { display: flex; justify-content: space-between; align-items: center; gap: 16px; } .sidebar { flex: 0 0 240px; } /* fixed 240px */ .main { flex: 1; } /* takes remaining space */
Main axis vs Cross axis: When flex-direction: row (default), main axis is horizontal → justify-content controls horizontal alignment, align-items controls vertical. Flip when flex-direction: column.
Chapter 9

CSS Grid

Grid Container Properties

Set display: grid on the parent.

PropertyDescription
grid-template-columnsDefines column track sizes. e.g. 1fr 1fr 1fr or repeat(3, 1fr).
grid-template-rowsDefines row track sizes.
gap / column-gap / row-gapSpace between grid cells.
justify-itemsAligns items horizontally within their cell. Default stretch.
align-itemsAligns items vertically within their cell.
justify-contentAligns the entire grid horizontally within the container.
align-contentAligns the entire grid vertically within the container.
.grid { display: grid; grid-template-columns: repeat(3, 1fr); gap: 24px; } /* Auto-fit responsive grid */ .auto-grid { display: grid; grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)); gap: 16px; }

Grid Item Properties

PropertyDescription
grid-columnShorthand for start/end column line. e.g. 1 / 3 spans columns 1–2. 1 / -1 spans full width.
grid-rowShorthand for start/end row line.
grid-column: span 2Span 2 columns from current position.
justify-selfOverride horizontal alignment for a single item.
align-selfOverride vertical alignment for a single item.
.featured { grid-column: 1 / -1; /* spans full width */ } .sidebar { grid-column: span 1; grid-row: 1 / 3; /* spans 2 rows */ }
Grid vs Flexbox: Use Grid for two-dimensional layouts (rows and columns). Use Flexbox for one-dimensional layouts (a row or a column). They work great together — Grid for page structure, Flexbox for component internals.
Chapter 10

Responsive Design

Media Queries

Media queries apply styles only when certain conditions are met — most commonly screen width.

/* Mobile-first: base styles apply to all sizes */ .container { padding: 1rem; } /* Tablet and up */ @media (min-width: 640px) { .container { padding: 2rem; } } /* Desktop and up */ @media (min-width: 1024px) { .container { max-width: 1200px; margin: 0 auto; } }
Mobile-first: Write base styles for small screens, then use min-width media queries to add complexity for larger screens. This is easier to maintain and results in better performance on mobile.

Common Breakpoints

NameMin-widthTarget
sm640pxLarge phones, small tablets
md768pxTablets
lg1024pxLaptops / small desktops
xl1280pxDesktops
2xl1536pxLarge desktops

Responsive Images & Fluid Widths

/* Images never overflow their container */ img { max-width: 100%; height: auto; } /* Fluid container with max width */ .container { width: 100%; max-width: 1200px; margin: 0 auto; padding: 0 1rem; }
Chapter 11

CSS Custom Properties (Variables)

Defining & Using Variables

Custom properties (CSS variables) let you store values and reuse them throughout your stylesheet. Defined with --name, used with var(--name).

/* Define on :root to make globally available */ :root { --color-primary: #2563eb; --color-text: #1a1a18; --radius: 8px; --spacing-md: 1rem; } button { background: var(--color-primary); border-radius: var(--radius); padding: var(--spacing-md); } /* Fallback value */ p { color: var(--color-text, #333); }
CSS variables are live — changing a variable with JavaScript (el.style.setProperty('--color-primary', 'red')) instantly updates all elements that use it. Great for theming.
Chapter 12

Transitions & Animations

Transitions

Transitions smoothly animate a change from one CSS value to another (triggered by state changes like :hover).

PropertyDescription
transition-propertyWhich CSS property to animate. Use all or a specific property.
transition-durationHow long the transition takes. e.g. 300ms, 0.3s.
transition-timing-functionease (default), linear, ease-in, ease-out, ease-in-out, cubic-bezier(…).
transition-delayWait before starting. e.g. 100ms.
button { background: navy; transition: background 200ms ease, transform 150ms ease; } button:hover { background: royalblue; transform: translateY(-2px); }

Keyframe Animations

Animations run automatically without a state change trigger. Define keyframes with @keyframes, then apply with animation.

PropertyDescription
animation-nameName of the @keyframes rule.
animation-durationHow long one cycle takes.
animation-iteration-count1 (default), a number, or infinite.
animation-directionnormal, reverse, alternate, alternate-reverse.
animation-fill-modeforwards keeps end-state after finishing; backwards applies from-state during delay.
@keyframes fadeIn { from { opacity: 0; transform: translateY(10px); } to { opacity: 1; transform: translateY(0); } } .card { animation: fadeIn 300ms ease forwards; }

Transform

transform moves, scales, rotates, or skews an element visually without affecting layout.

FunctionEffect
translateX(n) / translateY(n)Move horizontally / vertically. Also translate(x, y).
scale(n)Scale uniformly. scale(1.05) = 5% larger.
rotate(deg)Rotate. e.g. rotate(45deg).
skewX(deg)Skew along the X axis.
Prefer transform and opacity for animations — they are GPU-accelerated and do not trigger layout recalculation. Avoid animating width, height, margin, or top/left.

Use overflow-x and overflow-y to control each axis independently.

Property / ValueSpace in layoutPointer events

Stacking Context & z-index

Elements are stacked on a z-axis. z-index controls which elements appear on top.

  • Only works on elements with position other than static, or flex/grid children.
  • Higher z-index = closer to the viewer (appears on top).
  • A new stacking context is created by position + z-index, opacity < 1, transform, filter, etc. Elements inside a stacking context are stacked relative to each other, not the whole page.
