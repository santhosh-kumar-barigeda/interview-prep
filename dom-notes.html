Chapter 1

What is the DOM?

The Document Object Model

The DOM (Document Object Model) is a programming interface created by the browser when an HTML page loads. It represents the page as a tree of objects — each HTML element becomes a node that JavaScript can read, modify, add, or remove.

  • The DOM is not JavaScript — it is a Web API provided by the browser.
  • JavaScript uses the DOM to make pages dynamic and interactive.
  • Any change to the DOM is immediately reflected in the browser rendering.
DOM vs HTML: HTML is the source text. The DOM is the live, parsed, in-memory tree the browser builds from that HTML. They can diverge — JavaScript and CSS can change the DOM without touching the original HTML file.

The DOM Tree

Every part of an HTML document is a node in the tree. Nodes have types:

Node typeConstantExample
Element node1<div>, <p>, <img>
Text node3The text content inside an element
Comment node8<!-- comment -->
Document node9The root document object itself
DocumentType node10<!DOCTYPE html>
// document is the entry point to the entire DOM typeof document; // "object" document.nodeType; // 9 (Document node) document.documentElement; // <html> element document.head; // <head> element document.body; // <body> element

Node Relationships

Every node knows its relatives. These properties let you navigate the tree:

PropertyReturns
node.parentNodeThe immediate parent node.
node.childNodesA live NodeList of all child nodes (including text/comment nodes).
node.childrenHTMLCollection of child element nodes only (no text/comment).
node.firstChild / lastChildFirst/last child node (any type).
node.firstElementChild / lastElementChildFirst/last child that is an element.
node.nextSibling / previousSiblingAdjacent sibling node (any type).
node.nextElementSibling / previousElementSiblingAdjacent sibling element only.
Prefer children over childNodes — childNodes includes text nodes (whitespace between tags), which is usually not what you want. Use children to get only element nodes.

Traversal Methods

Sometimes it is more convenient to find an element relative to one you already have, rather than querying from document:

const item = document.querySelector(".list-item"); item.parentElement; // direct parent element item.closest(".container"); // walks UP — first ancestor matching selector item.matches(".list-item.active"); // true/false — does this element match? // closest() is perfect for event delegation document.addEventListener("click", e => { const btn = e.target.closest("button"); // find clicked button even if child was clicked if (btn) handleClick(btn); });

CSS Classes

The classList API is the modern, safe way to manage CSS classes on an element. Avoid manipulating className directly as string concatenation is fragile.

MethodDescription
el.classList.add("a", "b")Add one or more classes.
el.classList.remove("a")Remove a class.
el.classList.toggle("a")Add if absent, remove if present. Returns new state (boolean).
el.classList.toggle("a", force)Add if force is true, remove if false.
el.classList.contains("a")Check if class exists. Returns boolean.
el.classList.replace("a", "b")Replace one class with another.
const el = document.querySelector(".menu"); el.classList.add("open"); // adds class el.classList.remove("hidden"); // removes class el.classList.toggle("active"); // flips active on/off el.classList.contains("open"); // true el.classList.replace("open", "closed"); // swap // Toggle based on condition el.classList.toggle("disabled", !isEnabled);

Inline Styles

The style property on an element maps to its inline style attribute. Property names are camelCased versions of CSS properties. Only use this for dynamic style changes — prefer toggling classes otherwise.

const box = document.querySelector(".box"); // Set inline styles — camelCase CSS properties box.style.backgroundColor = "#ff0000"; box.style.width = "200px"; box.style.transform = "translateX(50px)"; // Remove an inline style (reverts to stylesheet value) box.style.backgroundColor = ""; // Read computed (final) style — includes stylesheet rules, not just inline const computed = getComputedStyle(box); computed.getPropertyValue("background-color"); // "rgb(255, 0, 0)" computed.fontSize; // "16px"
getComputedStyle vs .style: el.style only reads inline styles (set directly on the element). getComputedStyle(el) returns the final calculated style after all stylesheets and inline overrides are applied — use this to measure real values.

Inserting Elements — Modern API

MethodDescription
parent.append(...nodes)Insert at the end. Accepts elements or strings. No return value.
parent.prepend(...nodes)Insert at the beginning.
el.before(...nodes)Insert just before el (as sibling).

insertAdjacentHTML / insertAdjacentElement

insertAdjacentHTML lets you inject HTML strings at precise positions relative to an element — without replacing or re-parsing existing content:

Position stringWhere it inserts
"beforebegin"Before the element itself (as previous sibling).
"afterbegin"Inside the element, before its first child.
"beforeend"Inside the element, after its last child.
"afterend"After the element itself (as next sibling).
const card = document.querySelector(".card"); card.insertAdjacentHTML("beforeend", ` <footer class="card-footer">Updated just now</footer> `); card.insertAdjacentHTML("afterbegin", '<span class="badge">New</span>');
innerHTML vs insertAdjacentHTML: Setting innerHTML destroys and recreates all children — losing event listeners and focus. insertAdjacentHTML adds content without touching existing nodes. Prefer insertAdjacentHTML when you only want to add, not replace.

Cloning Nodes

const original = document.querySelector(".card"); // cloneNode(true) — deep clone: element + all descendants const copy = original.cloneNode(true); copy.querySelector("h2").textContent = "Cloned card"; document.body.append(copy); // cloneNode(false) — shallow: only the element, no children const emptyDiv = original.cloneNode(false);
Chapter 5

Events

addEventListener

The modern way to listen for events. An element can have multiple listeners for the same event type — unlike the old onclick property which only allows one.

const btn = document.querySelector("button"); // addEventListener(type, handler, options) btn.addEventListener("click", function(event) { console.log("clicked", event); }); // Arrow function — but loses `this` reference to the element btn.addEventListener("click", (e) => console.log(e.target)); // Named function — can be removed later with removeEventListener function handleClick(e) { console.log("handled"); } btn.addEventListener("click", handleClick); btn.removeEventListener("click", handleClick); // must be same ref // Options object btn.addEventListener("click", handleClick, { once: true, // auto-remove after first call passive: true, // never calls preventDefault — improves scroll perf capture: false, // bubble phase (default) });

The Event Object

Every event handler receives an event object as its first argument, containing details about what happened:

Property / MethodDescription
e.typeEvent type string: "click", "keydown", etc.
e.targetThe element that triggered the event (where it originated).
e.currentTargetThe element the listener is attached to. Always equals this in a regular function handler.
e.preventDefault()Stop the browser's default action (e.g. form submit, link navigation).
e.stopPropagation()Stop the event from bubbling up (or capturing down) the DOM tree.
e.stopImmediatePropagation()Stop all other listeners on the same element too.
e.key / e.codeFor keyboard events: the key value ("Enter") and physical key ("KeyA").
e.clientX / e.clientYMouse position relative to the viewport.
e.pageX / e.pageYMouse position relative to the whole document (includes scroll).
document.querySelector("form").addEventListener("submit", (e) => { e.preventDefault(); // stop page reload const data = new FormData(e.target); console.log(Object.fromEntries(data)); }); document.addEventListener("keydown", (e) => { if (e.key === "Escape") closeModal(); if (e.key === "Enter" && e.ctrlKey) submit(); });

Event Bubbling & Capturing

When an event fires, it travels in three phases:

PhaseDirectionDescription
1. CapturingWindow → TargetEvent travels down from window to the target. Listeners added with capture: true fire here.
2. TargetAt the elementEvent reaches the element it fired on.
3. BubblingTarget → WindowEvent travels back up. Most listeners use this phase (default).
// Click on inner <button> inside <div> // Bubbling order: button → div → body → html → document → window div.addEventListener("click", e => { console.log("div heard click from:", e.target.tagName); // e.target = button (origin), e.currentTarget = div (this listener) }); // Stop bubbling — div won't hear it btn.addEventListener("click", e => { e.stopPropagation(); });
Not all events bubble: focus, blur, scroll, load do NOT bubble. Use their bubbling equivalents (focusin, focusout) or attach listeners to the target directly.

Event Delegation

Event delegation means attaching one listener to a parent instead of many listeners on each child. It works because events bubble. This is more efficient, and automatically handles dynamically-added children.

// ❌ Naive — one listener per item (expensive, misses dynamic items) document.querySelectorAll(".item").forEach(item => { item.addEventListener("click", handleClick); }); // ✅ Delegation — one listener on the parent const list = document.querySelector("ul"); list.addEventListener("click", (e) => { const item = e.target.closest(".item"); // find the clicked .item if (!item) return; // click was outside any .item handleClick(item); });

Chapter 7

DOM Performance

Reflow & Repaint

Every time you change the DOM or CSS, the browser may need to:

OperationTriggersCost
Reflow (layout)Size / position changes: width, height, font-size, adding/removing elements.Expensive — recalculates positions of potentially many elements.
RepaintVisual changes without layout: color, background, visibility.Cheaper than reflow, but still has cost.
Compositetransform, opacity changes (GPU-accelerated).Cheapest — happens on GPU, does not trigger reflow or repaint.
Avoid layout thrashing — reading a layout property (like offsetWidth) immediately after writing a style forces the browser to flush pending layout changes synchronously. Batch reads together and writes together.

DocumentFragment

A DocumentFragment is a lightweight, off-screen container. Build multiple elements inside it, then insert the fragment in one operation — triggering only one reflow instead of one per element.

// ❌ Causes a reflow on every append (slow) const ul = document.querySelector("ul"); items.forEach(item => { const li = document.createElement("li"); li.textContent = item; ul.appendChild(li); // touches live DOM each time }); // ✅ Build off-screen, then one single insert const fragment = document.createDocumentFragment(); items.forEach(item => { const li = document.createElement("li"); li.textContent = item; fragment.appendChild(li); // off-screen — no reflow }); ul.appendChild(fragment); // one reflow here

requestAnimationFrame

requestAnimationFrame(cb) schedules a callback to run just before the browser's next paint. Use it for animations and any DOM changes that should be visually smooth — it syncs with the display refresh rate (typically 60fps).

// Animate a box moving right let x = 0; function animate() { x += 2; box.style.transform = `translateX(${x}px)`; if (x < 300) requestAnimationFrame(animate); // continue loop } requestAnimationFrame(animate); // start // Cancel if needed const id = requestAnimationFrame(animate); cancelAnimationFrame(id);
Prefer CSS transitions/animations for simple visual effects — the browser can optimise them with the GPU. Use requestAnimationFrame for JavaScript-driven animations that depend on runtime data or physics.

Chapter 9

Observers

MutationObserver

A MutationObserver watches a DOM node for changes and fires a callback whenever the DOM is mutated. It is async — callbacks run as microtasks after the mutations have already happened.

const target = document.querySelector("#feed"); const observer = new MutationObserver((mutations) => { mutations.forEach(mutation => { if (mutation.type === "childList") { console.log("Children changed:", mutation.addedNodes); } if (mutation.type === "attributes") { console.log("Attribute changed:", mutation.attributeName); } }); }); observer.observe(target, { childList: true, // watch for added/removed children subtree: true, // watch all descendants, not just direct children attributes: true, // watch attribute changes characterData: true // watch text content changes }); observer.disconnect(); // stop observing

IntersectionObserver

An IntersectionObserver fires a callback when an element enters or leaves the viewport (or a specified root). It is far more efficient than listening to scroll events for this purpose.

// Lazy-load images as they enter the viewport const imgObserver = new IntersectionObserver((entries, obs) => { entries.forEach(entry => { if (!entry.isIntersecting) return; const img = entry.target; img.src = img.dataset.src; // swap in the real src obs.unobserve(img); // stop watching once loaded }); }, { threshold: 0.1, // fire when 10% of the element is visible rootMargin: "0px 0px -50px 0px" // shrink detection zone by 50px at bottom }); document.querySelectorAll("img[data-src]").forEach(img => { imgObserver.observe(img); });
IntersectionObserver vs scroll event: Scroll listeners fire hundreds of times per second and require manual position calculations. IntersectionObserver callbacks are throttled by the browser and only fire on actual visibility changes — much more efficient.
Chapter 10

Useful Patterns

DOMContentLoaded vs load

// DOMContentLoaded — HTML parsed, DOM ready. Images may still be loading. document.addEventListener("DOMContentLoaded", () => { initApp(); // safe to query and manipulate the DOM here }); // load — everything loaded (images, stylesheets, fonts, iframes) window.addEventListener("load", () => { console.log("Page fully loaded"); }); // If script has defer attribute, DOMContentLoaded fires after all deferred // scripts run — so initApp() can go directly at the bottom of a deferred script.

Debounce & Throttle

Two techniques to control how often a handler runs on high-frequency events like scroll, resize, or input:

Debounce

Wait until the user stops firing the event for N ms, then run once. Good for search inputs — only search after the user pauses typing.

Throttle

Run at most once every N ms regardless of how many events fire. Good for scroll/resize where you need periodic updates but not every pixel.

// Debounce implementation function debounce(fn, delay) { let timer; return (...args) => { clearTimeout(timer); timer = setTimeout(() => fn(...args), delay); }; } // Throttle implementation function throttle(fn, limit) { let last = 0; return (...args) => { const now = Date.now(); if (now - last >= limit) { last = now; fn(...args); } }; } // Usage search.addEventListener("input", debounce(handleSearch, 300)); window.addEventListener("scroll", throttle(updateNav, 100));

Custom Events

You can create and dispatch your own events to communicate between components without tight coupling:

// Create a custom event with optional data payload const event = new CustomEvent("cart:updated", { detail: { itemCount: 3, total: 49.99 }, bubbles: true, // let it bubble up the DOM cancelable: true // allow preventDefault() }); // Dispatch it from any element document.querySelector("#cart").dispatchEvent(event); // Listen for it anywhere up the tree document.addEventListener("cart:updated", (e) => { updateCartBadge(e.detail.itemCount); });
