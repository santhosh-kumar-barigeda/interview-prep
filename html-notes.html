Introduction to HTML

What is HTML?

HTML stands for HyperText Markup Language. It is the standard language used to create and structure content on the web.

  • HyperText – text that contains links (hyperlinks) to other resources.
  • Markup Language – uses tags to define and organize the structure of content (not logic or behavior).

The first release (1990) supported only text and hyperlinks. Over time, elements like images, forms, video, and semantic tags were added.

Markup vs Programming language: A markup language (HTML) defines structure. A programming language (JavaScript, Python) creates logic, functionality, and behavior.

Chapter 2

Document Anatomy

Basic Document Structure

<!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8" /> <meta name="viewport" content="width=device-width, initial-scale=1.0" /> <title>Page Title</title> </head> <body> <h1>Hello World</h1> <p>This is a paragraph.</p> </body> </html>
Head element

Contains meta-information — not visible on the page. Holds <title>, <meta>, <link>, <style>, <script>.

Body element

Contains all visible page content — text, images, links, forms. Holds <h1>, <p>, <div>, <img>, <a>, etc.

DOCTYPE Declaration

DOCTYPE (Document Type Declaration) tells the browser which version of HTML the page uses, so it can parse and render it correctly.

<!DOCTYPE html> ← HTML5 declaration (always first line)

Without DOCTYPE: browsers enter "quirks mode" — they may misrender the page, break layouts, cause SEO issues, and make debugging harder.

Script Loading

The <head> loads before the <body>. If a script tries to manipulate DOM elements before they exist, it will fail.

MethodWhereBehavior
End of <body>Before </body>All HTML is loaded first — safest classic approach
deferIn <head>Script downloads in parallel; runs after HTML is parsed. Recommended modern approach.
asyncIn <head>Downloads in parallel; runs immediately when ready — may run before DOM is ready. Use for independent scripts (e.g. analytics).
<script src="app.js" defer></script> ← modern best practice

Core Syntax & Structure

Elements & Tags

Anatomy of an HTML Element

<a href="https://example.com" target="_blank">Click here</a> ↑ opening tag ↑ attribute=value pairs ↑ content ↑ closing tag
  • Tag – the keyword inside angle brackets, e.g. <p>
  • Element – the full structure: opening tag + content + closing tag
  • Attribute – provides extra information in name="value" format inside the opening tag

Block & Inline Elements

Block elements — start on a new line and take up the full available width
ElementDefinition
Semantic Layout

<footer>Footer of a section or document.
<main>Dominant content of the page. Only one per page.
<nav>Section with navigation links.
<section>Groups related content that belongs together.
<article>Self-contained piece of content (blog post, article).
<aside>Content related to, but separate from, the main content.
Text & Content
<h1>–<h6>Heading elements from highest to lowest level.
<p>Paragraph of text.
<blockquote>Section quoted from another source.
<hr>Horizontal rule — thematic break. Void element.
Lists
<ul>Unordered list.
<ol>Ordered (numbered) list.
<li>List item.
Generic Container
<div>Generic container for grouping. No semantic meaning.
Inline elements — sit within a line of text, only take up as much width as their content needs
ElementDefinition
<span>Generic inline container. No semantic meaning.
<strong>Strong importance, displayed bold.
<em>Emphasis, displayed italic.
<a>Hyperlink.
<small>Smaller text, often used for fine print.
<label>Label for a form input.
<input>Form input field (replaced inline).
<img>Embeds an image (replaced inline).
<code>Snippet of code.
Nesting rule: Block elements can contain inline elements, but inline elements should not contain block elements. e.g., placing a <div> inside a <p> is invalid HTML.
Inline elements are of two types:

1. Non-replaced inline elements — depend on internal content. Whatever you write between the tags is what gets displayed. width and height have no effect.
Examples: <span>, <a>, <strong>, <em>

2. Replaced inline elements — the browser replaces the element with external content (image, form control) loaded via attributes like src or type. Because of this, width and height are respected.
Examples:
— <img> — loads an image file from src.
— <audio> / <video> — loads media and renders browser playback controls.
— <input> — renders a native UI control based on the type attribute.

Void Elements

Void Elements (Self-Closing / Empty Elements): Elements that have no closing tag and cannot have content inside them. In HTML5 the trailing /> is optional.
TermMeaning
Void elementOfficial HTML spec term. No children, no closing tag.
Self-closing elementInformal name for the same thing.
Empty elementInformal name — no content inside.

All void elements: <img>, <input>, <br>, <hr>, <meta>, <link>, <area>, <base>, <col>, <embed>

Semantic Elements

Semantic elements clearly describe their meaning to both the browser and the developer — the tag name tells you what the content is, not just how it looks.

Non-semanticSemantic equivalent
<div id="header"><header>
<div id="nav"><nav>
<div id="content"><main>
<div id="footer"><footer>
Why it matters:
— Accessibility: screen readers use semantic elements as landmarks.
— SEO: search engines weigh content inside semantic tags appropriately.
— Readability: your HTML is self-documenting — easier to maintain.

HTML Attributes

Attributes provide additional information about an element. Always written in the opening tag as name="value" pairs.

<a href="https://example.com" target="_blank">Click me</a>

Global Attributes

These attributes can be used on any HTML element.

AttributePurpose
idUnique identifier for an element. Used by CSS (#id) and JavaScript (getElementById). Must be unique per page.
classOne or more space-separated class names. Used by CSS (.class) and JavaScript. Multiple elements can share a class.
styleInline CSS styles applied directly to the element. Avoid in favour of external CSS.
titleAdvisory text shown as a tooltip on hover.
langDeclares the language of the element's content, e.g. lang="en".
hiddenHides the element from display. Equivalent to display: none.
tabindexControls keyboard tab order. 0 = natural order, -1 = remove from tab order.
<p id="intro" class="lead text-dark" data-section="hero">Hello</p>
What happens if two elements have the same id?

HTML does not throw an error — the page still renders. But the behaviour breaks:
— CSS: Both elements get styled, because CSS selects all matching elements regardless.
— JavaScript: getElementById returns only the first matching element and ignores the rest.
— Anchor links: #id in a URL scrolls to the first element only.

Duplicate ids are invalid HTML. Always keep them unique per page.

data-* Attributes

Custom attributes for storing extra information on any element. Name them data- followed by anything you want (e.g. data-user-id, data-theme). Read in JavaScript via element.dataset.userId — the browser auto-converts kebab-case to camelCase.

<!-- HTML --> <div data-user-id="42" data-theme="dark">...</div> <!-- JavaScript --> el.dataset.userId // "42" el.dataset.theme // "dark"

HTML Comments

Comments are notes in the HTML source that the browser ignores — they are never shown to the user.

Used to explain code, temporarily disable markup, or leave notes for other developers.

<!-- This is a comment --> <!-- TODO: replace this placeholder image --> <img src="placeholder.jpg" alt="placeholder" /> <!-- <p>This paragraph is disabled</p> -->
Comments are visible in "View Source" and browser DevTools — never put passwords or sensitive data in comments.

&copy;©Copyright symbol.

Nesting Rules

HTML has strict rules about which elements can go inside which. Violating them causes browsers to silently auto-correct your markup, often in unexpected ways.

RuleExample
Block elements can contain both block and inline elements<div> can hold <p>, <h2>, <span>
Inline elements should only contain inline elements or text<span> should not contain <div> or <p>
<p> cannot contain block elementsPutting <div> inside <p> auto-closes the <p>
<a> cannot nest another <a>Links cannot be inside links
<ul> / <ol> direct children must be <li>Cannot put <p> directly inside <ul>
<table> rows must be inside <tbody> / <thead> / <tfoot><tr> directly in <table> is auto-wrapped in <tbody>
<li> must be inside <ul> or <ol>A bare <li> outside a list is invalid
Tip: Use the browser's DevTools (Elements panel) or an HTML validator (validator.w3.org) to catch nesting errors. Browsers silently fix them, so the page may look fine while the HTML is actually invalid.

HTML Elements in Depth

Head Elements

Everything inside <head> is metadata — not visible on the page, but essential for how the browser and search engines handle it.

ElementPurpose
<title>Sets the page title shown in the browser tab and in search engine results. Required on every page.
<link>Links external resources. Most commonly used to attach a CSS stylesheet via rel="stylesheet". Void element.
<style>Embeds CSS directly in the HTML document. Useful for small pages; prefer external stylesheets for larger projects.
<script>Embeds or links JavaScript. Use defer or async in <head>, or place before </body>.
<base>Sets a base URL for all relative URLs in the document. Rarely used. Void element.

Meta Tags — <meta>

Void element. Provides metadata: charset, viewport, description, author, robots.

Meta tagPurpose
charset="UTF-8"Tells the browser to use UTF-8 encoding — correctly displays characters from all languages.
viewportMakes page responsive — scales correctly on mobile devices.
descriptionSummary shown by search engines in results (important for SEO).
authorSpecifies the author of the document.
robotsControls search engine crawling: index/noindex, follow/nofollow.
<head> <meta charset="UTF-8" /> <meta name="viewport" content="width=device-width, initial-scale=1.0" /> <title>My Page</title> <link rel="stylesheet" href="style.css" /> <script src="app.js" defer></script> </head>

Headings — <h1> to <h6>

ElementWhen to use
<h1>Main page title. Use only once per page — highest importance for SEO.
<h2>Major section headings on the page.
<h3>Sub-sections within an <h2>.
<h4>Sub-sections within an <h3>. Rarely needed.
<h5>Sub-sections within an <h4>. Almost never used.
<h6>Lowest heading level. Deepest nesting, practically unused.
Never skip heading levels — don't jump from <h1> to <h4>. Screen reader users navigate the page by headings, so the hierarchy must make logical sense.

Beyond <div> and <span>, HTML provides elements that carry meaning about the content they hold.

ElementPurpose
<header>Introductory content or navigational links for its section or page.
<nav>A set of navigation links. Used as a landmark by screen readers.
<main>The dominant content of the page. Only one per page.
<section>A thematic grouping of content, typically with a heading.
<article>Self-contained content that could be distributed independently (blog post, card).
<aside>Content tangentially related to the surrounding content (sidebar, callout).
<footer>Footer for its nearest section or the document.
<time>A specific time or date. Use datetime="2025-01-15" for machine-readable format.
<address>Contact information for the nearest <article> or <body>.
<mark>Highlights text as relevant in the current context. Yellow highlight by default.
<abbr>Marks an abbreviation. Use title attribute for the full form — shown as tooltip on hover.
<details>Disclosure widget — content hidden by default, shown when clicked to expand.
<summary>The visible label for a <details> element. Must be its first child.
<blockquote>A long quotation from another source. Use cite attribute for the source URL.
<cite>The title of a referenced creative work (book, film, article).
<time datetime="2025-06-01">June 1, 2025</time> <abbr title="HyperText Markup Language">HTML</abbr> <mark>highlighted text</mark> <details> <summary>Click to expand</summary> <p>Hidden content shown on expand.</p> </details>

Links & Navigation

Links are created with the <a> (anchor) element. The href attribute defines where the link goes.

Link typeSyntaxWhen to use
Absolute URLhref="https://example.com"Linking to an external website. Always includes the full protocol and domain.
Relative URLhref="about.html"Linking to pages/files within the same site. Relative to the current file's location.
In-page anchorhref="#section-id"Jumps to an element on the same page that has a matching id.
Email linkhref="mailto:hello@example.com"Opens the user's email client with the address pre-filled.
Phone linkhref="tel:+1234567890"On mobile devices, tapping the link opens the dialler.
Download linkhref="file.pdf" downloadPrompts the browser to download the file instead of navigating to it.
<!-- External link --> <a href="https://example.com" target="_blank" rel="noopener noreferrer">Visit Example</a> <!-- In-page anchor --> <a href="#about">Go to About section</a> <section id="about">...</section> <!-- Email & phone --> <a href="mailto:hello@example.com">Email us</a> <a href="tel:+1234567890">Call us</a>
target="_blank" security: Opening links in a new tab gives the new page access to the opener via window.opener. Always add rel="noopener noreferrer" to prevent this.

Navigation with <nav>

Wrap site navigation links in <nav> to give them semantic meaning. Screen readers use <nav> as a landmark. Use aria-current="page" on the active link.

<nav aria-label="Main navigation"> <ul> <li><a href="/" aria-current="page">Home</a></li> <li><a href="/about">About</a></li> <li><a href="/contact">Contact</a></li> </ul> </nav>

Lists

ElementPurpose
<ul>Unordered list — bullet points. Use when order doesn't matter.
<ol>Ordered list — numbered. Use when sequence matters (steps, rankings).
<li>A list item. Must be a direct child of <ul> or <ol>.
<dl>Description list — for term/definition pairs.
<dt>The term in a description list.
<dd>The description or definition for the preceding <dt>.
<!-- Unordered --> <ul> <li>HTML</li> <li>CSS</li> </ul> <!-- Ordered --> <ol> <li>Install Node</li> <li>Run npm install</li> </ol> <!-- Description list --> <dl> <dt>HTML</dt> <dd>HyperText Markup Language</dd> </dl>

Tables

Use tables for tabular data — data that naturally fits rows and columns. Never use tables for page layout.

ElementPurpose
<table>Wrapper for the entire table.
<caption>Title/description for the table. Placed as the first child of <table>. Good for accessibility.
<thead>Groups the header rows. Used for the column titles row.
<tbody>Groups the main data rows.
<tfoot>Groups footer rows (e.g. totals). Rendered after <tbody> regardless of position in HTML.
<tr>A single table row.
<th>Header cell — bold and centred by default. Use scope="col" or scope="row" for accessibility.
<td>Data cell. Accepts colspan and rowspan.
<table> <caption>Monthly Sales</caption> <thead> <tr><th scope="col">Month</th><th scope="col">Revenue</th></tr> </thead> <tbody> <tr><td>January</td><td>$4,000</td></tr> <tr><td>February</td><td>$5,200</td></tr> </tbody> <tfoot> <tr><td>Total</td><td>$9,200</td></tr> </tfoot> </table>
colspan / rowspan:
— colspan="2" on a <td> makes it span 2 columns.
— rowspan="2" makes it span 2 rows.
Both reduce the number of <td> elements needed in affected rows/columns.

Forms

A form collects user input and sends it to a server or handles it with JavaScript.

ElementPurpose
<form>Wrapper for all form controls. action = where to send data; method = get or post.
<input>Single-line input field. Behaviour controlled by type attribute. Void element.
<textarea>Multi-line text input. Size controlled by rows and cols attributes.
<button>Clickable button. type="submit" submits the form; type="button" for JS handlers; type="reset" clears fields.
<select>Dropdown list. Wraps <option> elements. Add multiple to allow multi-select.
<option>A single choice inside a <select>. Use selected to pre-select, value to set the submitted value.
<label>Links descriptive text to an input via for="inputId". Clicking the label focuses the input. Important for accessibility.
<fieldset>Groups related form controls visually and semantically with a border.
<legend>Caption for a <fieldset>. Must be the first child inside <fieldset>.
<form action="/submit" method="post"> <fieldset> <legend>Personal Info</legend> <label for="name">Name</label> <input type="text" id="name" name="name" required /> <label for="bio">Bio</label> <textarea id="bio" name="bio" rows="4"></textarea> <select id="role" name="role"> <option value="dev">Developer</option> <option value="design" selected>Designer</option> </select> </fieldset> <button type="submit">Send</button> </form>
Input types: text, email, password, number, date, checkbox, radio, file, range, color, submit, reset, hidden

Media

ElementPurpose
<img>Embeds an image. Void element. Requires src and alt.
<audio>Embeds audio. Use controls to show playback UI. Supports nested <source> elements for fallback formats.
<video>Embeds video. Use controls, width, height, poster (thumbnail). Supports nested <source> elements.
<source>Specifies a media resource for <audio> or <video>. Browser picks the first format it supports. Void element.
<picture>Provides multiple image sources for responsive/art-direction. Wraps <source> elements + a fallback <img>.
<figure>Groups media content with its caption. Semantically self-contained.
<figcaption>Caption for a <figure>. Can appear before or after the media.

Chapter 5

Accessibility

Why Accessibility Matters

Accessibility (a11y) ensures your HTML can be used by everyone — including people using screen readers, keyboard-only navigation, or assistive technologies.

  • ~15% of the world's population has some form of disability.
  • Accessible sites are better for SEO — semantic structure helps search engines too.
  • Many countries have legal requirements for web accessibility (WCAG standards).

Semantic HTML as Accessibility

The single most effective accessibility improvement is using the correct semantic element. Screen readers use elements like <nav>, <main>, <header>, and <footer> as landmarks — users can jump directly between them.

  • Use <button> for buttons, not <div> — it gets keyboard focus for free.
  • Use <a> for navigation, <button> for actions.
  • Use proper heading order — <h1> → <h2> → <h3> — never skip levels.
  • Every form <input> must have an associated <label> via for="inputId".

ARIA Attributes

ARIA (Accessible Rich Internet Applications) attributes supplement HTML semantics when a native semantic element isn't available or sufficient.

AttributePurpose
aria-labelProvides an accessible name for an element with no visible text label, e.g. an icon button: <button aria-label="Close">.
aria-hidden="true"Hides decorative elements from screen readers (e.g. icon SVGs, decorative images).
roleAdds ARIA role when a semantic element isn't available, e.g. role="button" on a <div> acting as a button.
aria-current="page"Marks the currently active link in a navigation. Screen readers announce it as the active link.
<!-- Decorative image — screen readers skip it --> <img src="divider.png" alt="" /> <!-- Icon button with accessible name --> <button aria-label="Close menu">✕</button> <!-- Div acting as a button --> <div role="button" tabindex="0">Click me</div>
First rule of ARIA: Don't use ARIA if a native HTML element can do the job. A <button> is always better than <div role="button">.

Keyboard Navigation

All interactive elements must be reachable and operable via keyboard alone.

AttributePurpose
tabindex="0"Makes a non-interactive element (e.g. <div>) focusable and adds it to the natural tab order.
tabindex="-1"Removes an element from the tab order, but allows programmatic focus via JavaScript (element.focus()).
Native interactive elements — <a>, <button>, <input>, <select>, <textarea> — are all focusable by default. You only need tabindex for custom interactive components built from non-interactive elements.
