Chapter 1

Introduction to React Native

What is React Native?

React Native is an open-source framework by Meta for building native mobile apps (iOS and Android) using React and JavaScript. Unlike a web view wrapper, React Native maps React components to real native UI controls — you get genuine native performance and look-and-feel, not a browser inside an app.

FeatureDetail
LanguagesJavaScript / TypeScript
UI outputReal native widgets (not HTML/CSS)
PlatformsiOS, Android, Web (via React Native Web), Windows, macOS
Logic threadJS runs on a separate thread; UI thread stays free
ToolingExpo (managed) or React Native CLI (bare)

React vs React Native

Renders toDOM (HTML elements)Native views (UIView, View)
StylingCSS / TailwindStyleSheet API (JS objects)

Hooks / StateSame React APISame React API
All React concepts — components, props, state, hooks, context — work identically in React Native. The only difference is the host environment: native views instead of the DOM.

How React Native Works

React Native uses a bridge (old architecture) or JSI (new architecture, React Native 0.71+) to communicate between the JavaScript thread and the native UI thread.

ThreadResponsibility
JS ThreadRuns your React/JS code — component logic, state, hooks, business rules.
Native / UI ThreadRenders native views, handles gestures and animations.
Shadow ThreadCalculates layout using Yoga (a C++ Flexbox engine) before applying to native.
The New Architecture (Fabric + JSI + TurboModules) removes the async bridge bottleneck, enabling synchronous JS ↔ native calls and faster rendering. Enabled by default from React Native 0.76.

Chapter 2

Core Components

View & Text

View is the fundamental building block — equivalent to a <div>. It supports layout, style, and touch handling. All text must be inside a Text component — you cannot render a string directly inside a View.

import { View, Text, StyleSheet } from "react-native"; function Card({ title, subtitle }) { return ( <View style={styles.card}> <Text style={styles.title}>{title}</Text> <Text style={styles.subtitle}>{subtitle}</Text> </View> ); } const styles = StyleSheet.create({ card: { padding: 16, backgroundColor: "#fff", borderRadius: 12 }, title: { fontSize: 18, fontWeight: "600", color: "#1a1a18" }, subtitle: { fontSize: 14, color: "#888", marginTop: 4 }, });

Image

Image renders local or remote images. You must always specify width and height — unlike the web, images have no intrinsic size in React Native.

import { Image } from "react-native"; // Local asset <Image source={require("./assets/logo.png")} style={{ width: 100, height: 100 }} /> // Remote URL <Image source={{ uri: "https://example.com/avatar.jpg" }} style={{ width: 64, height: 64, borderRadius: 32 }} resizeMode="cover" />

TextInput

TextInput is the native text input. Use value + onChangeText for controlled inputs. Common props: placeholder, keyboardType, secureTextEntry, multiline, autoCapitalize.

import { TextInput, useState } from "react-native"; function SearchBar() { const [query, setQuery] = useState(""); return ( <TextInput value={query} onChangeText={setQuery} placeholder="Search…" keyboardType="default" autoCapitalize="none" style={{ borderWidth: 1, borderRadius: 8, padding: 10 }} /> ); }

TouchableOpacity & Pressable

React Native has no onClick on View. Use Pressable (recommended, flexible) or TouchableOpacity (dims on press) to make anything tappable.

import { Pressable, TouchableOpacity, Text } from "react-native"; // Pressable — fine-grained pressed/hovered/focused states <Pressable onPress={() => handleSubmit()} style={({ pressed }) => ({ opacity: pressed ? 0.7 : 1 })} > <Text>Submit</Text> </Pressable> // TouchableOpacity — simple opacity feedback <TouchableOpacity onPress={() => handlePress()} activeOpacity={0.7}> <Text>Tap me</Text> </TouchableOpacity>

ScrollView & FlatList

ScrollView renders all children at once — fine for small lists. FlatList lazily renders only visible items and is the right choice for any list that can grow.

import { FlatList, Text, View } from "react-native"; function UserList({ users }) { return ( <FlatList data={users} keyExtractor={item => item.id.toString()} renderItem={({ item }) => ( <View> <Text>{item.name}</Text> </View> )} ItemSeparatorComponent={() => <View style={{ height: 1, backgroundColor: "#eee" }} />} /> ); }
Use FlatList for any dynamic list. Use SectionList for grouped lists (e.g. contacts A–Z). Never put a FlatList inside a ScrollView — their scroll conflicts.

Modal & SafeAreaView

Modal renders content above everything else. SafeAreaView automatically insets content to avoid the notch, home bar, and status bar on iOS and Android.

import { SafeAreaView, Modal, View, Text } from "react-native"; // SafeAreaView — always wrap root screen content <SafeAreaView style={{ flex: 1 }}> <YourScreen /> </SafeAreaView> // Modal <Modal visible={isOpen} animationType="slide" onRequestClose={() => setIsOpen(false)}> <View><Text>Modal content</Text></View> </Modal>
Chapter 3

Styling

StyleSheet API

Styles in React Native are JavaScript objects — there is no CSS. StyleSheet.create() validates your styles at dev time and optimises them by sending style IDs over the bridge instead of full objects.

import { StyleSheet, View, Text } from "react-native"; const styles = StyleSheet.create({ container: { flex: 1, backgroundColor: "#f5f4f0", padding: 16, }, heading: { fontSize: 24, fontWeight: "700", color: "#1a1a18", marginBottom: 8, }, }); function Screen() { return ( <View style={styles.container}> <Text style={styles.heading}>Hello</Text> </View> ); }

Flexbox in React Native

React Native uses Flexbox for layout — but with two key differences from the web: flexDirection defaults to column (not row), and flex: 1 means "take all remaining space in the parent".

PropertyWeb defaultReact Native default
flexDirectionrowcolumn
alignContentstretchflex-start
flexShrink10
// Fill screen, center content const styles = StyleSheet.create({ screen: { flex: 1, justifyContent: "center", alignItems: "center" }, row: { flexDirection: "row", gap: 8 }, });

Dynamic & Conditional Styles

Pass an array to style to merge multiple style objects. Later entries override earlier ones — useful for conditional or variant styles.

// Array syntax — merged left to right <View style={[styles.base, isActive && styles.active, { marginTop: 8 }]}> // Platform-specific values import { Platform } from "react-native"; const shadow = Platform.select({ ios: { shadowColor: "#000", shadowOpacity: 0.1, shadowRadius: 4 }, android: { elevation: 4 }, });
Chapter 4

Navigation

React Navigation

React Navigation is the standard navigation library for React Native. It provides Stack, Tab, and Drawer navigators that map to native navigation patterns on each platform.

NavigatorUse case
StackScreen-by-screen navigation with a back stack (e.g. list → detail).
BottomTabTab bar at the bottom — main app sections (Home, Search, Profile).
DrawerSlide-out sidebar menu.
NativeStackUses native OS navigation stack — better performance than JS Stack.

Stack Navigator Setup

import { NavigationContainer } from "@react-navigation/native"; import { createNativeStackNavigator } from "@react-navigation/native-stack"; const Stack = createNativeStackNavigator(); function App() { return ( <NavigationContainer> <Stack.Navigator initialRouteName="Home"> <Stack.Screen name="Home" component={HomeScreen} /> <Stack.Screen name="Detail" component={DetailScreen} /> </Stack.Navigator> </NavigationContainer> ); }

Navigating & Passing Params

Every screen receives a navigation prop. Use navigation.navigate() to go to a screen, navigation.goBack() to go back, and pass data via params.

// Navigate and pass params function HomeScreen({ navigation }) { return ( <Pressable onPress={() => navigation.navigate("Detail", { userId: "42", name: "Alice" }) }> <Text>Go to Detail</Text> </Pressable> ); } // Read params in destination screen function DetailScreen({ route }) { const { userId, name } = route.params; return <Text>{name} — {userId}</Text>; }

Expo Router

Expo Router is a file-based routing system (like Next.js) built on top of React Navigation. Each file in the app/ directory becomes a route automatically.

// app/index.tsx → "/" (home screen) // app/profile.tsx → "/profile" // app/posts/[id].tsx → "/posts/:id" (dynamic route) // Navigate programmatically import { router } from "expo-router"; router.push("/profile"); router.push({ pathname: "/posts/[id]", params: { id: "42" } }); router.back();
Chapter 5

Platform APIs & Device Features

Platform Module

Use Platform.OS to detect the current platform and write platform-specific code without separate files.

import { Platform } from "react-native"; // Conditional logic const isIOS = Platform.OS === "ios"; const isAndroid = Platform.OS === "android"; // Platform.select — returns the matching value const fontSize = Platform.select({ ios: 16, android: 14, default: 15 }); // Platform-specific file — Button.ios.tsx / Button.android.tsx // React Native auto-picks the right file based on platform

Secure Storage

expo-secure-store encrypts data using the device's secure enclave (iOS Keychain / Android Keystore). Use it for sensitive values — auth tokens, passwords, API keys. Values are tied to the app and the device; uninstalling the app deletes them.

import * as SecureStore from "expo-secure-store"; // Write — value must be a string await SecureStore.setItemAsync("auth_token", token); // Write an object — serialise first await SecureStore.setItemAsync("user", JSON.stringify({ id: "42", role: "admin" })); // Read const raw = await SecureStore.getItemAsync("user"); const user = raw ? JSON.parse(raw) : null; // Delete await SecureStore.deleteItemAsync("auth_token");
Never store sensitive data in AsyncStorage — it is unencrypted plain text on the device file system. Use expo-secure-store for tokens, credentials, and anything that should not be readable if the device is compromised.

Permissions

Accessing device features (camera, location, notifications) requires runtime permissions on both iOS and Android. Use the expo-permissions or react-native-permissions library to request them.

import * as Location from "expo-location"; async function getLocation() { const { status } = await Location.requestForegroundPermissionsAsync(); if (status !== "granted") { alert("Permission denied"); return; } const location = await Location.getCurrentPositionAsync({}); return location.coords; }

Animations with Animated API

The built-in Animated API drives smooth, performant animations by running them on the native thread via useNativeDriver: true. For complex animations, use Reanimated 3.

import { Animated, useRef, useEffect } from "react-native"; function FadeIn({ children }) { const opacity = useRef(new Animated.Value(0)).current; useEffect(() => { Animated.timing(opacity, { toValue: 1, duration: 400, useNativeDriver: true, // runs on UI thread — no JS overhead }).start(); }, []); return <Animated.View style={{ opacity }}>{children}</Animated.View>; }
